3. zip

示例1：并行循环

把“直接按索引循环”和 zip 写法放在一起对比一下，体会下差异。

索引版

问题：

  • 代码更长，多了 range(len(...))、索引、下标访问 这三件套。
  • 如果两列表长度不同，就要再写 min(len(names), len(scores))，否则容易越界。
  • 可读性差：看到 i 时，要先在脑子里把它翻译成“第 i 个人”。

zip 版

优点：

  • 一行写完，语义直接：把名字和分数“并排”取出，读到就能秒懂。
  • 天然安全：长度不一致时自动截断，不会抛 IndexError。
  • 可迭代对象不限于 list——tuple、set、生成器、range、甚至文件对象都能 zip，索引法就无能为力了。

额外好处：多列表并行

性能 & 习惯

  • zip 返回的是惰性迭代器，不会一次性生成大列表；索引切片会。
  • Python 社区约定俗成：并行迭代就用 zip，看到就秒懂；索引循环反而需要额外思考“会不会越界”。

一句话
“能 zip 就别索引”，代码更短、更安全、可读性更高。

4. 惰性迭代器

什么是“惰性”（Lazy）？”

在编程中，“惰性”指的是：只有当你真正需要某个值的时候，它才会被计算或生成，而不是一开始就全部准备好。

这就像你点外卖：

  • 非惰性（急切/eager）：厨师一口气做完所有菜，全端上来 → 占用桌面空间（内存）。
  • 惰性（lazy）：你吃一道，厨师做一道 → 按需供应，节省空间。

优点 说明

8. 生成器（Generators）与 yield

一、什么是生成器（Generator）？

生成器是一个“懒人版”的迭代器：它不会一次性把所有数据准备好，而是“你要一个，我才生成一个”。

简单类比：

想象你在吃自助餐：

  • 普通列表 → 厨师一口气做了 100 道菜全端上来（占桌子）
  • 生成器 → 你说“我要吃”，厨师才做一道给你（按需供应）

这样更节省空间（内存），特别适合处理大量数据！

特性 return yield

✅ 过程解析：

  1. 第一次 next()：运行到第一个 yield，输出“第一步”，返回 "A"，暂停
  2. 第二次 next()：从暂停的地方继续，输出“第二步”，返回 "B"，再暂停
  3. 第三次：继续往下走…
  4. 第四次：已经没有 yield 了 → 抛出异常

💡 for 循环会自动捕获 StopIteration，所以你不用手动处理。

9. 类型提示（Type Hints）& typing

一、类型提示（Type Hints）——>给变量和函数“加说明书”

  • 帮你提前发现错误（配合工具如 mypy）
  • 提高代码可读性，别人一看就知道该传什么
