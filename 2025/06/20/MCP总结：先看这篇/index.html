MCP是什么

MCP（Model Context Protocol，模型上下文协议）。这是一种开放标准，旨在统一大型语言模型（LLMs）与外部数据源和工具之间的通信协议。

MCP允许应用程序以一种标准化的方式向LLMs提供上下文信息，这使得不同来源的数据可以更容易地被集成到LLMs中。

此外，MCP也被描述为类似于USB-C端口的一种接口，对于AI应用来说，它提供了一种通用的方法来连接不同的数据源和服务。通过使用MCP，开发者能够创建更加灵活且功能丰富的AI驱动的应用程序，因为它们可以无缝地与多种外部服务进行交互。

通过 uv run weather.py 启动 MCP 工具服务，建立标准输入输出的通信，初始化 MCP 协议会话。

定义一个协程方法 connect_to_server，用于建立与 MCP 工具服务器的连接。因为涉及异步 I/O（比如启动子进程、读写流等），所以使用 async def。

这一行创建了一个 StdioServerParameters 对象，表示将以 stdio（标准输入输出）方式启动一个 MCP 工具服务进程。

参数解释：

  • command='uv'：表示用 uv 命令来运行程序（uv 是比 pip 更快的 Python 包/脚本运行工具）。
  • args=['run', 'weather.py']：实际执行的是 uv run weather.py，也就是运行你的 weather.py MCP 工具。
  • env=None：不传入额外环境变量，使用当前 Python 环境的默认变量。
  • stdio_client(server_params)：启动 MCP 工具服务（uv run weather.py），并将其标准输入输出连接包装为一个异步流（reader/writer）。
  • exit_stack.enter_async_context(...)：将其注册到 AsyncExitStack 中，确保后续退出时能自动清理（类似于 with 自动释放资源）。
  • await 是因为这个是异步上下文管理器。

最终结果 stdio_transport 是一个元组 (stdio, write)：

  • stdio 是 StreamReader，可以从 MCP 工具读数据；
  • write 是 StreamWriter，可以向 MCP 工具写入命令。

解包上述元组，赋值给本地变量 stdio 和 write。

使用 stdio 和 write 初始化一个 ClientSession，表示 MCP 协议层面的连接会话对象。

仍然使用 exit_stack 管理资源，退出时自动关闭。
