MCP番外：Cline与模型的交互过程

发表于 2025-06-19 更新于 2025-06-20 分类于 code 阅读次数： 本文字数： 4.1k 阅读时长 ≈ 15 分钟

前言

前面我们介绍了MCP基础：从原理到实战和MCP进阶：带你深入掌握MCP，对MCP协议的原理有了比较清晰的认识，明确了MCP协议主要规定的是如何发现和调用函数的，也明确了MCP协议并没有规定如何与模型进行交互。之所以这篇文章叫做番外篇也是出于MCP协议并没有规定如何与模型进行交互的考虑。在MCP进阶一文中提到MCP协议没有对模型的输入和输出格式进行进行要求，因此不同的MCP Host就可能使用不同的格式来与模型沟通，比如Cline用的就是XML，Cherry Studio用的是Function Call，本文将以Cline为例讲解MCP Host是如何与模型沟通的。

本文参考这里，主要内容包括：

  • 截获模型输入/输出，展示Cline与模型交互的具体细节
  • 如何构建Agent常用的ReAct模式以及Cline用的XML协议与ReAct模式之间的关系

目标：不是单纯的了解Cline的XML协议，而是通过学习这个协议了解如何编写一个像Cline一样能够与模型持续交互的程序。

Cline的XML协议与ReAct的关系

先以流程图的形式总结下上述流程：

文字版说明（让我们站在Cline的角度看待这个过程）：

  • 我问模型一个问题：纽约明天的天气怎么样？
  • 模型想了想，觉得要调用我这边某个工具才能得到答案，然后按固定格式告诉了我工具名和参数
  • 我去调用了工具，将结果给到模型
  • 模型拿到我给的工具结果，思考总结后将最终答案返回给我

那如果我的问题变一变呢，让其将结果写入到result.md中，那么在前会加一套调用别的工具的流程，之后才能完成任务：

其中：

  • <thinking>对应了思考（thought）
  • 调用工具/给出最终答案对应了行动（Action），紧挨在思考之后
  • 如果任务未完成，即模型需要使用我（Cline）这边的能力，那么模型就需要观察（Observation）我这边的反馈结果。

Thought，Action，Observation是ReAct中最重要的三个词，ReAct是一篇2022的论文（REACT: synergizing reasoning and acting in language models）提出的一个词，它是reasoning和acting两个单词的合体，他提出的ReACT理念可以在不需要人干预的前提下，让模型自主思考，自主调用各类外部工具，从而完成用户指定的任务，就像Cline一样，本质上其实是个Agent。Agent是一个能持续思考，持续调用外部工具，直至解决用户问题的程序。Cline本质上也是根据ReAct思想进行它的Agent流程的，注意ReAct是个思想，精髓在于Thought，Action，Observation这三个词，并不对交互格式做要求，无论你是像Cline一样用XML格式，还是像Cherry Studio一样用function call那种格式，甚至用ReAct论文中那种近乎纯文本的格式，都是ReAct思想。
