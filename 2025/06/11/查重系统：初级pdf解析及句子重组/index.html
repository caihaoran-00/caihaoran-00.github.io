引入所需库

  • streamlit: 构建网页前端界面。
  • pdfplumber: 读取 PDF 文件的表格和文字。
  • Simhash: 文本相似度初筛。
  • TF-IDF: 关键词相似度计算。
  • sentence_transformers: 深度语义向量模型，用于高阶语义比对

Streamlit 页面配置

  • 设置网页标题和布局宽度

模型加载

  • 加载适用于中文的 BERT 句向量模型（text2vec-base-chinese）。
  • 使用 Streamlit 的缓存机制避免重复加载
条件 不加缓存 加了 @st.cache_resource
模型加载 每次都重新加载 只加载一次，后续直接复用
性能 慢、浪费资源 快、节省计算资源
使用场景 临时实验 实际部署时强烈推荐加缓存

✅PDF文件单元提取：

  • 从 PDF 文件中提取所有文本内容（包括表格和非表格部分），并标注其页码和位置信息（bounding box）
  • filter(...)用于过滤所有空字符串
  • ✅outside_bbox() 会返回去除了表格区域的页面副本对象

查重逻辑模块

深度学习方法（BERT语义相似度）

  • 对提取到的文本单元进行过滤（长度大于设定值）

  • 通过 Sentence-BERT 编码成向量

  • 使用 semantic_search 计算每段文字之间的语义相似度（余弦）

  • 提取相似度超过阈值的匹配项

  • hits 是个列表：每一项是 filtered_units1[i] 匹配到的 filtered_units2[j] 及其得分

  • matched_indices_in_doc1: 用来统计有多少句子被匹配，用于后续计算重复率

  • hit_list[0] 是最相似的匹配

  • 重复率定义为：被匹配的单元数 ÷ 总有效单元数

  • matched_indices_in_doc1 记录了多少句子被成功匹配

  • 为方便前端展示或分析，将匹配结果按相似度从高到低排序。

传统方法（SimHash + TF-IDF）
  • 使用 SimHash 先快速筛除不相似的段落
  • 对于疑似相似的，再使用 TF-IDF + cosine_similarity 精确比对
  • 记录每对相似文本及其相似度分数

高亮匹配项生成图片

  • 给指定页的指定区域画上红框，并保存为 PNG 图片
  • 用于可视化展示匹配文本位置
