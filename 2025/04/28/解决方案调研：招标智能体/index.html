解决方案调研：招标智能体

发表于 2025-04-28 分类于 theory 阅读次数： 本文字数： 622 阅读时长 ≈ 2 分钟

需求

  1. 网页自动爬虫智能体：从各大公共交易平台提取项目招标网页数据和下载各类招标信息附件；

  2. 数据提取智能体模型：下载下来的各类招标信息附件(PDF、word、excel、图片等)进行文件解析并按关键词把非结构化数据转换成结构化数据，经清洗校验后存入公司数据库，如：项目编号、项目名称、工程概况、招标概况、开标日期、开标地点、招标控制价等；

  3. 投标报价预测智能体模型:利用历史招标数据对类拟项目进行模拟推算，预测投标报价中标概率；

需求分析与大致方案

这是个非常典型但也很有挑战的 三阶段智能体系统，分别是 数据抓取 → 数据加工 → 数据建模与推理。

总体架构

三个智能体模块将以模块化方式设计，彼此独立但通过数据流和接口协作。整体流程如下：

  1. 网页自动爬虫智能体：从公共交易平台抓取招标网页数据和附件，存储原始数据；
  2. 数据提取智能体：解析附件，提取关键字段，清洗并结构化后存入数据库；
  3. 投标报价预测智能体：基于历史数据和提取的结构化数据，训练预测模型，输出中标概率。

1. 网页自动爬虫智能体

目标：从各大公共交易平台抓取招标网页数据，并下载附件。

核心步骤：

  • 平台选定：明确目标平台（如全国公共资源交易平台、各省市平台等），收集URL规则。
  • 爬虫开发：
    • 网页结构分析：用浏览器开发者工具（F12）分析HTML结构，找到数据位置。
    • 爬取技术选型：
      • 静态页面：直接用 requests + BeautifulSoup 抓取。
      • 动态渲染页面：用 Selenium、Playwright 之类模拟浏览器操作。
    • 附件下载：
      • 找到附件链接并批量下载。
      • 附件类型（PDF/Word/Excel/图片）统一存储，文件夹按日期/来源分类。
  • 异常处理和稳定性设计：
    • 自动重试机制。
    • 反爬虫识别与应对（如IP代理池、User-Agent伪装、限速）。
    • 日志记录（爬取成功/失败的日志）。

建议技术栈：

  • Python：requests，BeautifulSoup4，Selenium，Playwright
  • 调度系统：Airflow（定时任务管理，后期可扩展）
