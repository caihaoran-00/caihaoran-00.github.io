你的作业是进一步阅读关于生成式人工智能的内容，并尝试识别一个目前尚未应用生成式人工智能的领域，思考如何将其应用到该领域。与传统方式相比，影响会有何不同？你是否能够实现以前无法做到的事情，或者是否更高效？撰写一篇300字的总结，描述你梦想中的人工智能初创公司，并包括以下标题：“问题”、“我将如何使用人工智能”、“影响”，以及可选的“商业计划”。

最后是知识检查环节：

关于大型语言模型，以下哪些是正确的？

  1. 每次都会得到完全相同的回答。
  2. 它能完美地完成任务，非常擅长计算、生成可用代码等。
  3. 回答可能会因相同的提示而有所不同。它也非常擅长为你提供某些内容的初稿，无论是文本还是代码。但你需要对结果进行改进。

答案：3，大型语言模型是非确定性的，回答会有所变化，但你可以通过调整温度设置来控制其变化程度。同时，你不应期望它能完美完成任务，它的作用是为你完成繁重的工作，通常意味着你会得到一个不错的初稿，然后需要逐步改进。

探索与比较不同的大语言模型 (LLMs)

1. 核心概念辨析：地基与建筑

  • 基础模型 (Foundation Models) vs. 大语言模型 (LLMs)
    • 基础模型是一个更广泛的概念。它们是大规模的、经过预训练的、具有强适应性的模型，通常是多模态的（处理文本、图像、音频等）。它们是构建下游 AI 应用的“地基”。
    • LLMs 是基础模型的一个子集，专门专注于语言处理。
    • 关键点： 所有 LLM 都是基础模型，但并非所有基础模型都是 LLM（例如纯视觉模型）。

2. 模型的分类与功能

根据输入输出的不同，模型被分为几类：

  • 文本/代码生成模型： 输入文本，输出文本或代码。例如 GPT-4, Llama 2, Mistral。
  • 图像生成模型： 输入文本提示，生成图像。例如 DALL-E, Stable Diffusion。
  • 嵌入模型 (Embeddings)： 输入文本，输出数字向量。这是机器理解语义的关键，用于搜索、聚类和 RAG（检索增强生成）系统。

3. 决策十字路口：开源 vs. 专有

在选择模型时，这是第一个主要决策点：

特性 开源模型 (Open Source) 专有模型 (Proprietary)
代表例子 Llama (Meta), Falcon, Mistral, Hugging Face 上的模型 GPT-4 (OpenAI), Claude, Gemini
优点 透明性（代码可见）、控制权（可微调权重）、隐私（可本地部署）、通常免费使用许可。 易用性（API 接入即用）、高性能、维护省心、安全性由厂商负责。
缺点 维护成本高（需自建基建）、更新慢、需自己负责安全性修补。 黑盒（无法知晓内部运作）、数据隐私顾虑、成本（按 Token 付费）。

4. 部署模式：服务 vs. 模型

决定了模型类型后，如何运行它？

  • 模型即服务 (Model as a Service / MaaS)：
    • 像使用水电一样，通过 API 调用云端的模型（如 Azure OpenAI Service）。
    • 优势： 自动扩缩容、无需管理 GPU 服务器、快速集成。
  • 自托管模型 (Model as a Platform/Artifact)：
    • 下载模型文件（Weights），在自己的服务器或 Docker 容器中运行。
    • 优势： 极致的控制权。
    • 劣势： 需要专业的运维团队和昂贵的硬件支持。

5. 探索工具：Azure AI Studio

视频中详细展示了微软的工具平台，用于辅助上述决策：

  • Model Catalog（模型目录）： 一个聚合中心，可以浏览 OpenAI、Meta、Mistral 等各家厂商的模型。
  • Model Card（模型名片）： 类似于食品营养成分表，详细列出模型的训练数据、适用场景、局限性和许可证信息。
  • Benchmarks（基准测试）： 提供可视化的数据图表，对比不同模型在准确率、推理速度、连贯性上的表现。

6. 提升模型表现的“金字塔”策略

当你选定一个模型后，它可能还不能完美解决你的问题。课程提出了一个从易到难的优化路径（这也是本课程后续章节的伏笔）：

  1. 提示工程 (Prompt Engineering) [最底层/最简单]：
    • 通过优化输入的 Prompt（提供上下文、Few-shot 示例、设定角色语气）来引导模型。
    • 适用场景： 绝大多数通用任务，成本最低。
  2. 检索增强生成 (RAG) [中层]：
    • 当模型需要知道它训练数据之外的信息（如公司内部文档、最新新闻）时使用。
    • 利用向量数据库检索相关信息，并将其注入到 Prompt 中。
    • 适用场景： 智能客服、企业知识库助手。
  3. 微调 (Fine-tuning) [高层]：
    • 使用特定的数据集对模型进行再训练，更新其权重。
    • 适用场景： 需要特定的输出格式、学习独特的语言风格、或因为对延迟要求极高而无法使用超长 Prompt 时。
  4. 从头训练 (Training from Scratch) [顶层/最难]：
    • 通常不建议初创公司尝试，需要海量数据和算力资源。

7. 总结

  • 互补性： 上述技术不是互斥的，而是互补的。实际生产中，最强大的系统往往是 RAG + 微调 + 提示工程 的组合。
  • 迭代： 选择模型不是一次性的工作，而是一个“选择 -> 测试 (Benchmarking) -> 优化”的持续迭代过程。
