管网Agent需求梳理

发表于 2025-11-17 更新于 2026-01-31 分类于 code 阅读次数： 本文字数： 8.2k 阅读时长 ≈ 30 分钟

前言

这是一个需求梳理文档，暂名：管网Agent。是一个关于自动生成HAZOP文档的Agent。本文将重点梳理整个工作流以及各步骤的输入输出。

正文

**明确目标：**从头到为的所有操作都是为了生成/提取一些段落/短句，并将这些段落/短句放到HAZOP模版文档的固定位置，以自动化的生成HAZOP文档。

HAZOP预分析

web入口（基本信息+资料上传与解析）

需要选择/填写报告基本信息包含：

  • 项目（下拉列表，内置几种项目，每个项目绑定介质和站场功能），例：华东地区/上海分公司/站场1/风险分析项目
  • 会议日期，例：2025.6.17
  • 报告展示名称，例：桂林输气站危险与可操作性（HAZOP）分析
  • 参会人员，例：张三、李四、王五

需要上传站场资料信息包含：

  • 节点划分图纸，例：HAZOP分析节点划分依据图纸.pdf
  • 站场功能、建设及改造历史、周边环境，例：
  • 主要系统资料，
  • 主要工艺流程资料，
  • 主要设备和技术参数资料，
  • 主要工艺参数和保护参数资料，

报告基本信息的四项的作用

  • 项目字段用于绑定介质和站场功能（这两个字段用于后续流程）
  • 其余三个字段用于填充在HAZOP报告的固定位置

各站场资料信息作用

  • 节点划分图纸中通常包含节点名称（比如节点1：XXXX，节点2：XXXX…），也可能包含工艺流程（位于图纸左下），主要设备（上方）

  • 站场功能、建设及改造历史、周边环境 以及 主要系统资料作为附加信息（类似于背景信息）用于后续的提示词工程，这部分不直接出现在HAZOP报告中

  • 主要工艺流程资料，例：SPC-0202PR01-0.pdf第9页的3.2 工艺流程，按三级标题+内容形式进行输出

  • 主要设备和技术参数资料，例：SPC-0202PR01-0.pdf第10页的表 3.3-1 站场设备配置一览表

  • 主要工艺参数和保护参数资料，例：关键工艺管道设备系统明细表及技术参数.docx（输出是啥样的？）

节点管理

所有原料准备好了，下面进行节点管理，由节点划分图纸解析得到的节点数量决定这里有几个sheet页，每页对应一个节点。

本页展示（以节点一为例，一共有三个sheet页，因为有三个节点）：

  • 项目展示名称，例：风险分析项目（新增）
  • 会议日期，例：2025.6.17
  • 节点号，例：1（新增）
  • 参考图纸，例：桂林输气站工艺流程图（新增，=节点划分图纸的文件名称）
  • 参会人员，例：张三、李四、王五
  • 装置/系统，例：进出站及越站返输（新增，= 节点1名称）

这6项是节点工作表的表头。↓

  • 工艺流程，下拉列表（新增）
  • 操作维护，下拉列表（新增）
  • 设备，下拉列表（新增）

这三项是操作人员/用户进行选择，用于维护节点的标签项。

下面的标签/字段都是可以手动输入或AI生成的：

  • 节点描述

    上传文档的界面选择项目时即关联了介质，再加上本页面的工艺流程，操作维护，设备字段，使用这四个字段去1-节点描述汇总.xls匹配词条，通过匹配到的词条/节点描述 + （待考究）已经解析得到的主要设备信息、主要流程去让大模型生成节点描述。

  • 设计意图

    思路同上，2-设计意图汇总.xls

  • 主要设备及技术参数

    思路同上，4-主要设备及参数汇总.xls，这里只通过工艺流程去匹配即可。通过匹配到的词条 加上 识别到的所有设备名称，让大模型给这些设备按节点（的工艺流程）分类。

  • 主要连锁控制点

    • BPCS/DCS联锁
    • SIS联锁
    • 报警及人员响应
    • 其他联锁（如设备本体联锁，火气联锁等）
    • 机械保护（物理保护）

    西一线长铝末站 进站过滤分离流程 天然气 正输/分输/过滤分离 正输流程 进站压力PT1101低压(3.3MPa)低低（3.2MPa）报警;压力高（5.1MPa）,高高（5.3MPa）报警 压力变送器 压力变送器 压力传感器 进站压力PT**低压(**MPa)低低（**MPa）报警;压力高（**MPa）,高高（**MPa）报警

    对于匹配到的词条/行，需要的列是

    • 报警（详细）
    • BPCS/DCS 联锁
    • 其他保护
    • SIS 联锁
    • 物理保护（泄放装置）

    如果报警列有内容，将其内容与前面提取到的保护措施进行匹配，并将其放到报警及人员响应框；

    而BPCS/DCS 联锁、其他保护、SIS 联锁、物理保护（泄放装置）的内容需与联锁动作、联锁设备拼接后做为完整内容去与前面提取到的保护措施进行匹配，并纳入到对应框中；

场景管理

使用介质、站场功能、工艺流程、操作维护、设备标签去5-场景列表.xlsx文件中匹配相似的词条（场景），可按标签手动引用或添加场景，添加场景后将刚才确认引用的词条的内容放到以下的对应位置：

  • 选择（复选框）

  • 序号，例：2.1.1

  • 参数，例：压力（来自词条匹配）

  • 偏差，例：压力高（词条匹配）

  • 原因，例：来气压力高（词条匹配）

  • L，例：3（根据原因的内容查表，表待提供）

  • 后果，例：流量计出口压力大，对站内管道冲击大（词条匹配）

  • S，例：P:-，F:-，E:-，R:1（词条匹配）

  • R1，例：I（通过风险矩阵表计算）

  • 保护措施/修正条件，例：报警及人员相应[0.1]: 1. 桂林输气站转供超声波流量计(DN300)精度范围：177m³/h-7200m³/h [0.1] …

    （这里是通过介质、站场功能、工艺流程、操作维护、设备等标签在场景列表中匹配到的词条，使用LLM从节点管理中的主要联锁/控制点（应该叫做保护措施）中选择的词条，带PFD，操作员手动添加条件修正【感觉应当是手工填在主要联锁/控制点中，而不是直接手工填在场景的保护措施/修正条件中】。）

  • R2（计算得到）

  • 建议措施（这里不填）

  • R3（不需计算）

业务理解

前提：目前使用产品原型上的站场进行实验，未做其他格式文件以及其他站场的兼容。

总体上来看，HAZOP预分析的步骤是：

  1. 资料解析
  2. 节点管理
  3. 场景管理

资料解析指的（目前关注，包括不限于）是：

  • 从节点图纸解析出节点信息、流程信息、设备信息

    现阶段从文档（广西公司桂林输气站HAZOP报告（O版）.pdf）中截图作为输入，使用Qwen2.5VL-32B，节点信息准确，其他信息准确度不足。

  • 工艺流程/主要设备及参数

    从SPC-0202PR01-0.pdf中解析得到，均准确

  • 保护措施参数

    从关键工艺管道设备系统明细表及技术参数.docx解析得到，准确。

节点管理指通过上述资料解析得到的内容+节点名称+工艺流程+操作维护+设备，得到：

  • 节点描述

    短，不带位号

  • 设计意图

    详细，带位号

  • 主要联锁/控制点

  • 主要设备及技术参数

主要联锁/控制点

相当于工厂的保镖，

  1. 控制点 (Control Point)：
    • 就像你把燃气灶的火调大调小。
    • 在工厂里：这就是“调节阀”或“PID控制”。比如：为了保持水箱水位，阀门自动开大一点或关小一点。它的目的是让生产平稳进行。
  2. 联锁点 (Interlock Point)：
    • 就像电热水壶水烧干了，它会“咔哒”一下自动断电。这救了你的命，防止干烧起火。
    • 在工厂里：这就是“SIS系统”或“ESD系统”。比如：进站压力太高了（快爆炸了），系统强制关闭进站阀门。它的目的是保命、保设备。

一个完整的“点”必须包含三要素：

  1. 眼睛（检测元件）：谁在看？（比如：PT-101 压力变送器）
  2. 大脑（设定值/逻辑）：什么时候动手？（比如：压力 > 9.9 MPa 时）
  3. 手（执行动作）：动手干什么？（比如：关闭 进站阀 ESDV-101）

R1 & R2计算

变量介绍：

  • L：事故发生概率等级（应根据典型偏差原因概率表填入，目前直接使用检索回的场景条目中的L）
  • S：事故后果严重程度等级（根据现场分析人工填入，目前直接使用检索回的场景条目中的S）
  • R1：初始风险等级（根据风险评估矩阵计算）
  • R2：现有措施降级后风险等级（根据保护措施分类的对应PFD值，自动计算将险后的风险等级）

R1计算：

  • L乘以S列的每项数值
  • 从风险评估矩阵中找到对应风险等级
  • 取最大值。

例：

L=3，S={P : 1, F : 2, E : 3, R : 4}

  • 识别S中最大的值→4

  • 查找风险评估矩阵（见下图红色圈）→得到：Ⅲ

    即R1的值为Ⅲ。

R2计算：

每条保护措施对应PFD值（来自于节点管理页面默认/手动填入的小数），可用于降初始概率等级；

这里的保护措施是主要联锁/控制点的子集；主要联锁/控制点是资料解析-保护措施参数分条后的子集）

PFD值到0.1则降一级，到0.01降两级，到0.001降三级。比如PFD=0.7不降级，PFD=0.07降一级，PFD=0.007降两级；

x > 0.1 → 0级；
0.01 < x ≤ 0.1 → 1级；
0.001 < x ≤ 0.01 → 2级；
0.0001 < x ≤ 0.001 → 3级

同类保护措施中，所有的保护措施累计降险不可大于两级；

条件修正对应PFD可降后果严重程度等级

降低概率等级（L）或严重程度等级（S），最多降至1级，不可降至0级

降险后重新用风险矩阵，计算风险等级R2

保护措施降L，条件修正降S，同类保护措施最多降两级，最低降至1级。
