发表于 2025-07-15 分类于 code 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 12 分钟

前言

argparse是Python标准库中的一个常用模块，专门用来帮助轻松地编写用户友好的命令行接口程序。它是用来解析命令行参数的模块。在一个开源项目中看到了它的比较高阶的用法，索性开篇文章系统的整理下argparse模块的用法。本篇文章将从最基础的讲起，由浅及深的介绍argparse模块的使用，目标是仅看本篇文章甚至不用实操就能对argparse有个全面且具象的认识。

正文

首先，什么是argparse，argparse = arg + parse，

  • arg 是 argument（参数） 的缩写
  • parse 意思是“解析“

所以，argparse 就是用来 解析命令行参数的模块 。它是 Python 标准库中的一个模块，专门用来帮助你轻松地编写用户友好的命令行接口程序。

为什么需要 argparse？

在写 Python 脚本时，如果你希望这个脚本能通过命令行运行，并且接收一些输入参数，比如：

那你就会需要用到 argparse 来帮你处理这些参数。它可以：

  • 接收命令行参数
  • 自动生成帮助信息（如 -h 或 --help）
  • 做类型检查和默认值设置
  • 支持可选参数和位置参数
  • 让你的脚本更专业、更易于使用

第一部分

这是命令行的使用格式说明 ，告诉用户这个脚本应该怎么用。

  • demo1.py: 脚本名称
  • [-h]: 可选参数 -h 或 --help
  • -n NAME, --name NAME: 必须提供的参数（因为设置了 required=True）
  • [-a AGE]：可选参数，可以省略
  • [-ht HEIGHT]：可选参数，表示身高
  • [-v]：布尔型可选参数，出现就为 True
  • [-hb HOBBIES [...]]：可选参数，接收多个字符串
  • [-c {red,green,blue}]：可选参数，只能从指定值中选
  • {greet,farewell}：位置参数，只能是这两个值中的一个
  • target：位置参数，任意字符串

有几点需要注意：

  1. 虽然位置参数定义在了最前面，但在这里显示时是在最后面
  2. 被[]包着的参数是可选参数，未被包着的是必须提供的参数
  3. 必须提供的参数分为两种，一种是指定了required=True，另一种是位置参数
  4. 必须提供的参数一般不使用default参数
  5. 位置参数只是不带前缀--，其余用法相同

  1. {greet,farewell}

位置参数
所谓位置参数就是指定参数时不加前缀--，对位置敏感。

可选参数

有前缀--或-，--后面跟参数全称，-后面跟简称。

type

这里指定参数的数据类型，包括：str、int、float，注意：不包括bool，bool型参数可以使用：

指定改参数则为True，否则为False。

nargs

默认情况下，一个参数只接受一个值，nargs就是用来改变这个行为的。

  • +表示至少一个值，否则报错
  • *接收任意数量的值（包括 0 个）

default

该参数用于指定默认值，如果命令行未指定某参数，如果该参数定义默认值了就用该默认值替代，注意要与type类型一致，且一般不与required=True同时使用。

required

该参数用在可选参数中（位置参数本来就是必须指定的，所以不需要该参数），用于指定改参数必须指定，默认为False，所以一般出现时就是required=False。

choices

choices 的作用是：限制用户只能从指定的选项中选择一个值。 用中括号[]括起来，注意其中元素要与type匹配。

help

帮助信息，用于描述参数的意义。运行python demo1.py -h时会被显示出来。
