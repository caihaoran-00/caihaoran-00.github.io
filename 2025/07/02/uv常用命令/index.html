uv常用命令

发表于 2025-07-02 分类于 code 阅读次数： 本文字数： 1.2k 阅读时长 ≈ 4 分钟

前言

Python 开发中，虚拟环境和依赖管理是绕不开的话题。uv 是一个新兴但备受关注的工具，致力于加速并简化 Python 项目的构建、安装与管理流程。本文将介绍 uv 是什么，它与 Conda 的关系，以及一些实用的 uv 命令，助你轻松上手。

UV 是什么？

uv 是一个由 Astral 团队开发的 Python 包管理与构建工具，其主要特点包括：

  • 🚀 速度快：用 Rust 编写，比传统工具（如 pip）更快。
  • 📦 兼容性强：兼容 pip, virtualenv, pyproject.toml, requirements.txt 等现有生态。
  • 🧰 功能全：支持创建虚拟环境、安装依赖、项目初始化等一体化功能。
  • 🔒 可重现构建：依赖锁文件（uv.lock）保证环境一致性。

uv 并不是另起炉灶，而是借助现有生态的惯例与格式，提供了更高性能的替代方案，尤其适合现代 Python 项目管理。

UV 与 Conda 的关系

尽管 uv 和 conda 都提供依赖管理和环境隔离的功能，但它们的定位和底层设计差异明显：

项目 uv conda
实现语言 Rust C / Python
依赖来源 PyPI（使用 wheel 格式） Conda 自建的二进制包仓库
环境隔离 基于 venv 使用 Conda 自有环境系统
性能 极快（特别是依赖安装） 相对较慢
兼容性 更接近 Python 官方工具生态 生态独立，包兼容性不一定理想
应用场景 现代 Python 项目（特别是 Web/AI） 科学计算、大型多语言项目

简单来说，如果你主要面向 Python 项目开发，想要更快更轻量的依赖管理体验，uv 是一个比 conda 更现代化的选择；如果你需要管理 跨语言的科学计算环境，如 Python + R + Fortran，那 conda 依然是更成熟的方案。

❓在 Conda 环境中使用 uv 会怎么样？

  • uv 会优先使用 venv 创建独立虚拟环境，即使你当前已经处于 Conda 环境中。
  • 不建议混用：虽然技术上你可以在 Conda 环境中运行 uv 命令，但 uv 创建的 .venv 与 Conda 环境相互独立，不会共享依赖、解释器或路径。
  • 典型后果可能包括：
    • 你以为安装在 Conda 里，但实际装在 .venv；
    • 使用 uv run 时调用的是 .venv 的解释器，而不是 Conda 的；
    • 错误地依赖 Conda 中的库会导致 “ModuleNotFoundError”；

✅ 推荐做法：进入 Conda 环境时，不使用 uv 命令；使用 uv 时保持在系统解释器环境下（运行：conda deactivate，以退出conda环境），让它自行管理 .venv。

  • 通过uv pip install ...安装的包需使用uv pip uninstall...去卸载

总结

uv 是一个快速、现代、兼容性强的 Python 包管理工具。它的设计理念是替代 pip + venv + pip-tools 的组合方案，提供更统一且更快的开发体验。如果你已经习惯 Conda，也可以尝试将 uv 用于 Python-only 项目，享受更快的构建速度与更简洁的工作流程。
