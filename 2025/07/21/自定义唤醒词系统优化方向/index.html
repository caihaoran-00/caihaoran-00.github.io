自定义唤醒词系统优化方向

发表于 2025-07-21 分类于 code 阅读次数： 本文字数： 403 阅读时长 ≈ 1 分钟

前言

我们前面解析了自定义唤醒词的整体逻辑，对于该系统有了个整体的认识，结合代码原理以及初步测试结果，我们发现了目前系统的一些不足，基于此，在本文梳理目前的问题与优化方向，作为记录和分享。

正文

本文不涉及具体数据，只涉及测试总结和分析。由于系统处在相对早期状态，所以目前测试不涉及噪声环境或者远距离场景下的表现，只涉及相对普通的场景。测试人员测试发现目前该系统：

  • 唤醒率（召回率）/误拒绝率受说话人影响比较大，小孩和女人的误拒绝率很高
  • 对于男性测试者，误拒绝率虽然不高，但误接受率挺高

可见目前该系统处于相对早期的状态。由于本周市场那边将要进行拿去体验/拍视频等。所以目前最重要的是让小孩和女人能唤醒，怎么做呢？经验认为是训练素材的多样性不足，随机抽取了一部分训练素材进行试听，发现确实是这样，所以针对训练素材扩充进行实验，那么怎么扩充训练素材？

  • 换/增加TTS模块
  • 增加实际人声

显然换个TTS模块的成本低，就该方向进行实验，发现确实效果有大幅度提升。模型已给出，等待该次测试结果再做进一步实验。
