基本介绍

这个项目使用的是YOLO V8n这个小模型，包含两个训练文件（training.py对应的是粗训练,improve_training.py对应的是正经训练）使用的数据集来自这里（注意原项目中的标签是空的，这个问题我已修复），

数据集大小：

  • 训练集: 2218 张图片
  • 验证集: 382 张图片
  • 测试集: 89 张图片

数据集的分布：

  • 16298 个 飞溅 (Spatters)
  • 5686 个 气孔 (Porosity)
  • 2439 个 余高过量 (Excess Reinforcement)
  • 45 个 裂纹 (Crack)
  • 25 个 坏焊 (Bad Welding)
  • 22 个 好焊 (Good Welding)

可见数据存在较为明显的类别不均衡问题。但作为体验来说也无伤大雅。

mAP50 0.499 0.516 +3.4%

  • Box (P) = Precision（精确率）
  • Box (R) = Recall（召回率）
  • mAP50 = IoU=0.5 时的平均精度
  • mAP50-95 = IoU 从 0.5 到 0.95 的平均精度
  • IoU（Intersection over Union，交并比） 是目标检测任务中衡量预测框（Predicted Bounding Box）与真实框（Ground Truth）重合程度的核心指标。

**整体性能对比（all 行） **

mAP50 0.504 0.512 +1.6% 整体检测精度小幅提升
mAP50-95 0.297 0.302 +1.7% 严格指标也有改善
Precision 0.576 0.596 +3.5% 误检减少，预测更准
Recall 0.495 0.479 -3.2% ⚠️ 漏检略有增加

✅ 结论：优化策略提升了精确率和整体 mAP，但牺牲了少量召回率，说明模型变得更“保守”——宁可漏掉一些缺陷，也不愿误报。

**关键缺陷类别对比（重点关注） **

Crack（裂缝） mAP50 0.695 0.664 ❌ -4.5% 性能下降！ 裂缝是高危缺陷，此退化需警惕
Recall 0.707 0.619 ❌ -12.4% 漏检大幅增加，可能因增强策略或损失权重不当
Bad Welding mAP50 0.550 0.497 ❌ -9.6% 识别能力明显变差
Porosity（气孔） mAP50 0.464 0.531 ✅ +14.4% 表现显著提升，模型更擅长识别多数类
Spatters（飞溅） mAP50 0.443 0.549 ✅ +24.0% 改进效果最显著
Good Welding mAP50 0.647 0.589 ❌ -9.0% “良好焊缝”识别能力下降

估计主要原因还是前文提到的数据集分布不均衡。

项目目录及说明

主要文件和文件夹说明

  • README.md: 项目的核心文档，详细说明了项目的背景、如何安装依赖、以及如何运行训练和测试脚本。
  • requirements.txt: 定义了项目所需的所有 Python 依赖包，方便一键安装。
  • src/: 存放所有 Python 源代码的目录。
    • improve_training.py: 经过优化的主要训练脚本。
    • training.py: 项目的初始训练脚本，可看作是粗训练（预热）步骤。
    • run_inference.py: 用于推理和可视化的脚本，其结果保存在 outputs/inference_results/ 中。
  • outputs/: 存放所有输出结果的目录。
    • training_runs/: 存放 YOLOv8 训练过程的输出，每一个子文件夹代表一次完整的训练实验。
    • inference_results/: 存放 run_inference.py 生成的预测图和真实标签图。
  • data/: 存放所有数据集的根目录。
    • data/Weld/: 焊接缺陷数据集的核心文件夹，内含 train、valid 和 test 子集。
  • models/: 存放预训练模型权重文件的目录。
    • .pt 文件: yolov8n.pt 是从 Ultralytics 官方下载的预训练模型。best.pt (位于 outputs/training_runs/*/weights/ 目录下) 是我们自己训练出的最佳模型。
  • config/: 存放配置文件的目录。
    • custom_data.yaml: 数据集配置文件，告诉 YOLOv8 数据在哪里以及类别信息。
  • .venv/: Python 的虚拟环境文件夹。

训练参数说明

在 improve_training.py 中，model.train() 函数包含了大量的超参数，下面对这些参数进行详细说明：

参数 说明 作用
data 指向 data.yaml 配置文件的路径 定义训练、验证、测试数据集路径及类别信息
epochs 训练轮数 200 轮，增加轮数以获得更好的收敛效果
imgsz 输入图像的大小 640x640像素，平衡分辨率和显存使用
batch 每批处理的图像数量 8，为6GB显存显卡优化的批量大小
device 训练设备 指定使用GPU或CPU
workers 数据加载的CPU线程数 4，加速数据预处理和加载
project 训练结果保存项目根目录 指定输出目录为 outputs/training_runs
name 训练任务名称 保存为 ‘gpu_training_optimized’ 子目录
val 验证开关 启用，训练过程中同时进行验证
augment 数据增强开关 启用，提高模型泛化能力
cache 图像缓存策略 ‘disk’，硬盘缓存以获得确定性的训练
amp 混合精度训练 启用，使用 float16 + float32 提升训练速度和精度
lr0 初始学习率 0.001，训练起始时的学习率
lrf 最终学习率 0.01，训练结束时的学习率（相对于lr0的倍数）
cos_lr 余弦学习率衰减 启用，实现更平滑的学习率变化
optimizer 优化器类型 AdamW，具有更好的泛化性能
momentum 动量 0.937，SGD优化器的动量参数
weight_decay 权重衰减 0.0005，L2正则化防止过拟合
close_mosaic 关闭马赛克增强的轮次 10，最后10轮关闭马赛克增强
mosaic 马赛克增强 1.0，完全启用马赛克数据增强
mixup Mixup增强概率 0.1，混合两张图像进行数据增强
copy_paste 复制粘贴增强概率 0.1，随机复制粘贴对象进行数据增强
hsv_h HSV色相增强 0.015，微小的色相变化
hsv_s HSV饱和度增强 0.7，较大的饱和度变化
hsv_v HSV明度增强 0.4，中等程度的明度变化
degrees 随机旋转角度 10度，随机旋转图像
translate 随机平移比例 0.1，图像随机平移最大10%
scale 随机缩放比例 0.5，图像随机缩放范围为0.5倍
shear 随机剪切 0.0，不启用剪切变换
perspective 透视变换 0.0，不启用透视变换
flipud 垂直翻转概率 0.0，不启用垂直翻转
fliplr 水平翻转概率 0.5，50%的概率水平翻转
