🌱 一、什么是正则表达式？

正则表达式（regex）是一种描述字符串模式的工具，它可以用来：

  • ✅ 检查字符串是否符合某种格式（如邮箱、手机号）
  • ✅ 从文本中提取特定内容（如所有数字、URL）
  • ✅ 替换文本中的部分内容
  • ✅ 分割字符串（比 split() 更强大）

在 Python 中，我们通过内置的 re 模块来使用正则。

🔧 二、导入与基本语法

所有正则操作都通过 re 模块的函数完成。

📌 重要提示：使用原始字符串 r"" 来写正则，避免转义问题。

🛠️ 三、re 模块的核心函数

函数 作用 返回值
re.match() 从字符串开头匹配 Match 对象 或 None
re.search() 在整个字符串中找第一个匹配 Match 对象 或 None
re.findall() 找出所有匹配 列表
re.finditer() 返回匹配的迭代器 迭代器（每个是 Match 对象）
re.sub() 替换匹配内容 新字符串
re.split() 按正则分割字符串 列表

1. re.match(pattern, string) —— 从开头匹配

❌ 注意：如果开头不匹配，就返回 None

✅ 适用场景：验证输入是否以某模式开头（如手机号、身份证）。

2. re.search(pattern, string) —— 全局搜索第一个

✅ 即使不在开头也能找到。

✅ 适用场景：查找任意位置的第一个匹配。

3. re.findall(pattern, string) —— 找所有匹配

返回一个字符串列表，非常直观，适合初学者练习。

✅ 适用场景：提取所有符合条件的内容（如所有数字、邮箱）。

4. re.finditer(pattern, string) —— 返回迭代器

✅ 适合处理大文本，节省内存。

5. re.sub(pattern, repl, string, count=0) —— 替换

✅ 适用场景：脱敏、格式化、清理文本。

6. re.split(pattern, string) —— 正则分割

比 str.split() 更灵活。

2. 特殊字符转义

. * + ? ^ $ [] {} () | \ 有特殊含义，要匹配本身需转义：

3. 字符类 [...]

写法 含义
[aeiou] 匹配任意一个元音
[0-9] 匹配任意数字
[a-z] 小写字母
[A-Z] 大写字母
[a-zA-Z] 所有字母
[^0-9] 非数字（^ 在开头表示“非”）

4. 预定义字符类（快捷方式）

简写 等价于 含义
\d [0-9] 数字
\D [^0-9] 非数字
\w [a-zA-Z0-9_] 单词字符（字母、数字、下划线）
\W [^\w] 非单词字符
\s [ \t\n\r\f\v] 空白字符
\S [^\s] 非空白字符

5. 量词（重复次数）

量词 含义
* 0 次或多次
+ 1 次或多次
? 0 次或 1 次（可选）
{n} 恰好 n 次
{n,} 至少 n 次
{n,m} n 到 m 次

re.findall(r'ab?', 'a ab abb')括号中的ab?表示：

  • a：匹配字母 a
  • b?：匹配 0 个或 1 个 b（? 表示“前面的字符可有可无”）

所以，ab? 的含义是：

匹配 a，后面可以跟一个 b，也可以不跟

6. 锚点（位置匹配）

锚点 含义
^ 字符串开头
$ 字符串结尾
\b 单词边界（字母/数字/下划线 与 非这些字符之间）
\B 非单词边界

7. 分组 ( ... ) 与捕获

括号用于分组，并可以“捕获”内容。

✅ 适用场景：提取日期、电话区号、名字等结构化信息。

8. 非捕获组 (?: ... )

不想捕获？用 (?: ...)

找 http://... 或 https://... 的网址，但不要单独保存 http 或 https 这部分。

9. 或操作 |

⚠️ 注意优先级：^cat|dog 表示“以 cat 开头”或“包含 dog”，通常要加括号：

10. 非贪婪匹配（懒惰模式）

默认是贪婪的（尽可能多匹配），加 ? 变成非贪婪。

11. 零宽断言（高级）

不消耗字符，只检查“周围环境”。

写法 含义
(?=...) 正向先行断言（后面必须是 …）
(?!...) 负向先行断言（后面不能是 …）
(?<=...) 正向后行断言（前面必须是 …）
(?<!...) 负向后行断言（前面不能是 …）

⚙️ 五、修饰符（Flags）

通过 flags= 参数控制匹配行为。

修饰符 含义
re.I 或 re.IGNORECASE 忽略大小写
re.M 或 re.MULTILINE 多行模式：^ 和 $ 匹配每行开头结尾
re.S 或 re.DOTALL 单行模式：. 匹配换行符
re.X 或 re.VERBOSE 详细模式：允许写注释和空格

🧠 七、最佳实践

  1. 用 r"" 写正则：避免转义烦恼
  2. 先用 re.findall() 练习：直观看到结果
  3. 复杂正则用 re.VERBOSE：可读性更强
  4. 避免过度复杂：能用字符串方法就不用正则
  5. 测试工具：用 regex101.com（选 Python 模式）
