企业微信外部群消息获取调研

发表于 2026-05-14 分类于 code 阅读次数： 本文字数： 1.4k 阅读时长 ≈ 5 分钟

前言

前面我们做了HB.AI私域分型与督导系统技术方案和从Hermes Agent抽离WeCom能力：封装一个独立可复用的企业微信机器人接入项目，在实操时（API方式获取企业微信群消息时），发生了一些阻塞性的问题，本文记录碰到的问题以及调研别的解决方案。

外部群的写的是：使用微信或企业微信扫码加入

添加智能机器人

我们试下将智能机器人添加到群聊：

内部群是可以添加的，而外部群不能添加（是灰色的），xxx那个群已经添加过这个智能机器人了，所以也是灰的。

而智能机器人的API接入方式是我们需要的（获取群内at机器人的消息，然后经过LLM回复，再从智能机器人输出出去），并且当前项目的使用场景就是外部客户群。想想其它办法。

其他方式

既然不能实现外部群内添加智能机器人（企业微信的限制），那只能想想其他办法了。

首先想到的就是让客户下载个企业微信，成为企业员工，自然就可以拉内部群，也就可以添加智能机器人了；（用户摩擦较大）

结论

本文我们一起尝试了从企微外部群获取和发送消息，发现：

  1. 无法向企微外部群添加智能机器人（智能机器人才可以API方式调用）
  2. 会话内容存档+创建企业群发方式无法达到我们的需求，因为创建企业群发这个方式平均一天只能发一个消息
  3. 自动回复方式是枚举方式，不灵活，不满足需求
  4. 微信客服方式是1V1聊天，无法在企微外部群内聊，链接

当然还有一种方式是灰产的方式，使用真实的企微用户（比如就是你的账号）作为消息的收发点，通过逆向出来的API操作你的账号，我看的一个账号一年费用是￥960。链接就不放出来了。
