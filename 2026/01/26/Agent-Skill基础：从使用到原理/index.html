Agent Skill基础：从使用到原理

发表于 2026-01-26 更新于 2026-01-29 分类于 code 阅读次数： 本文字数： 5.3k 阅读时长 ≈ 19 分钟

前言

最近Agent Skill这个专业术语相当火爆，了解了下，还是Anthropic推出的行业标准。顾名思义，Agent Skill就是给Agent使用/赋予的技能，它的核心思想是渐进式披露（progressive disclosure），即按需取用/加载，而不是像工具调用或mcp那样把工具描述Schema完整放入系统提示词（导致即使问一个很简单的问题也会消耗大量token）。本文我们将一起看下Agent Skill的使用和原理（融入coze和opencode），最后还将比较Agent Skill与mcp的区别。

这就是一个Skill的文件结构，下面我们将介绍一下这个结构以及各文件内容（这是想手撸Skill的基础）：

  • 文件夹名称就是Skill的名字，必须为小写字母+连字符
  • SKILL.md是每个SKILL都包含的文件，其余文件可选
  • SKILL.md这个核心文件的结构固定分为两部分，且都是必须项。首先是YAML头部，用---包裹，必包含name和description字段（这里也包含了dependency字段，可能是coze独有的？不属于最通用的标准？），这部分是该SKILL的名片，会被放入到系统提示词中；第二部分就是Markdown主体了，这部分包含详细的工作流程、资源路径、注意事项和使用示例等，当触发这个SKILL后才会加载这部分Markdown主体，然后需要某些资源时（这里是supported-formats.md和convert_audio.py）再去加载对应资源。值得一提的是，对于.md文件资源会去读取具体内容，对于程序文件（这里是python文件），一般情况下不会去读取它的具体内容（直接运行），除非运行失败了，那么可能会去看下源码。
  • 每个SKILL的name和description信息会被放到系统提示词中，当这个技能被触发/选中才会去读取下方的Markdown主题，再进一步（需要时）才会去使用其他资源（其他md文件和程序文件），这种方式叫做渐进式披露，其实就是按需加载，这是Agent SKILL技术省token的核心手段
  • 对于资源文件，md文件放到references文件夹，程序文件放到scripts文件夹

16 个技能介绍

1. algorithmic-art（算法艺术）

使用 p5.js 创建算法艺术，支持种子随机化和交互式参数探索。用于生成艺术、流场或粒子系统。

2. brand-guidelines（品牌指南）

将 Anthropic 官方品牌颜色和排版应用到任何需要 Anthropic 外观和风格的产物中。当涉及品牌颜色、样式指南、视觉格式或公司设计标准时使用。

3. canvas-design（画布设计）

使用设计理念创建 .png 和 .pdf 格式的精美视觉艺术。当用户要求创建海报、艺术品、设计或其他静态作品时使用。创建原创视觉设计，避免复制现有艺术家作品以防版权侵权。

4. doc-coauthoring（文档协作）

引导用户完成文档协作的结构化工作流。当用户想要编写文档、提案、技术规范、决策文档或类似结构化内容时使用。此工作流帮助用户高效传达上下文、通过迭代完善内容，并验证文档是否适合读者。

5. docx（Word 文档处理）

全面的文档创建、编辑和分析工具，支持修订跟踪、评论、格式保留和文本提取。当 Claude 需要处理专业文档（.docx 文件）时使用：（1）创建新文档，（2）修改或编辑内容，（3）处理修订跟踪，（4）添加评论，或任何其他文档任务。

6. frontend-design（前端设计）

创建独特、生产级的高设计质量前端界面。当用户要求构建网页组件、页面、产物、海报或应用程序时使用（示例包括网站、落地页、仪表板、React 组件、HTML/CSS 布局，或美化任何网页 UI）。生成创意、精致的代码和 UI 设计，避免通用的 AI 美学。

7. internal-comms（内部沟通）

帮助我编写各种内部沟通内容的资源集，使用公司喜欢的格式。当被要求编写某种内部沟通内容时使用（状态报告、领导层更新、3P 更新、公司通讯、FAQ、事件报告、项目更新等）。

8. mcp-builder（MCP 构建器）

创建高质量 MCP（模型上下文协议）服务器的指南，使 LLM 能够通过精心设计的工具与外部服务交互。用于构建 MCP 服务器以集成外部 API 或服务，无论是 Python 还是 Node/TypeScript。

9. pdf（PDF 处理）

全面的 PDF 操作工具包，用于提取文本和表格、创建新 PDF、合并/拆分文档以及处理表单。当 Claude 需要填写 PDF 表单或以编程方式大规模处理、生成或分析 PDF 文档时使用。

10. pptx（PowerPoint 演示文稿）

演示文稿创建、编辑和分析。当 Claude 需要处理演示文稿（.pptx 文件）时使用：（1）创建新演示文稿，（2）修改或编辑内容，（3）处理布局，（4）添加评论或演讲者备注，或任何其他演示文稿任务。

11. skill-creator（技能创建器）

创建有效技能的指南。当用户想要创建新技能（或更新现有技能）以扩展 Claude 在专业知识、工作流或工具集成方面的能力时使用此技能。

12. slack-gif-creator（Slack GIF 创建器）

为 Slack 创建优化动画 GIF 的知识和工具。提供约束条件、验证工具和动画概念。当用户请求 Slack 动画 GIF 时使用，例如”为我制作一个 X 做 Y 的 Slack GIF”。

13. theme-factory（主题工厂）

使用主题对产物进行样式化的工具包。这些产物可以是幻灯片、文档、报告、HTML 落地页等。有 10 个预设主题，带有可以应用到任何已创建产物的颜色/字体，或即时生成新主题。

14. web-artifacts-builder（Web 产物构建器）

使用现代前端 Web 技术（React、Tailwind CSS、shadcn/ui）创建复杂多组件 claude.ai HTML 产物的工具套件。用于需要状态管理、路由或 shadcn/ui 组件的复杂产物——不适用于简单的单文件 HTML/JSX 产物。

15. webapp-testing（Web 应用测试）

使用 Playwright 与和测试本地 Web 应用程序的工具包。支持验证前端功能、调试 UI 行为、捕获浏览器截图和查看浏览器日志。

16. xlsx（电子表格处理）

全面的电子表格创建、编辑和分析工具，支持公式、格式化、数据分析和可视化。当 Claude 需要处理电子表格（.xlsx, .xlsm, .csv, .tsv 等）时使用：（1）创建带公式和格式化的新电子表格，（2）读取或分析数据，（3）修改现有电子表格同时保留公式，（4）电子表格中的数据分析和可视化，（5）重新计算公式。包含金融模型的颜色编码标准、数字格式规范、公式构建规则等专业要求。

小结1

好的，到这里我们通过coze 2.0实际体验了Agent Skill，对coze2.0的部分玩法和Agent Skill都有了基本的认识。

Agent Skill最大的优势就是渐进式披露机制使其相当省token（不用把所有文字都放入系统提示词了），其相当于SOP手册。

几乎所有能用工作流实现的效果都能通过Agent Skill实现，Agent Skill与MCP能成为行业标准的原因就是其可复用。

Agent Skill VS MCP

这了解Agent Skill的过程中，我想起了cline的运行方式（可参考这里)，cline的内置提示词来指导做事流程，采用XML方式实现工具调用（其实也是监控LLM输出，只不过不是function call那种输出方式），通过监测LLM的输出实现特定工具的调用。Agent Skill也是，通过提示词来指导LLM做事流程，然后需要执行某个程序脚本时就去执行某个脚本，但有个显著的不一样的点是渐进式披露（按需加载）机制，这极大的减少了token用量。

试想一下，如果提示词中的工具调用的定义很多（MCP中关于工具调用的定义也会被注入到系统提示词中），那么问一个简单问题也会消耗大量的token，且由于提示词过长，LLM的注意力也会受影响，这个问题可以被Agent Skill的渐进式披露机制解决。

那么既然Agent Skill的方式看起来比MCP更好，是不是MCP就要被完全替代/淘汰掉了呢？
