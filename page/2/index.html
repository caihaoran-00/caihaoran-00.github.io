发表于 2025-09-15 更新于 2025-09-17 分类于 code 本文字数： 3.4k 阅读时长 ≈ 12 分钟

前言

目前玩具的讲故事机制处于萌芽阶段，现在的方案是让LLM即兴创作，会有一个比较严重问题：生成的故事的AI味很重，无论什么故事感觉都一个套路，现在就这个问题进行优化，目前的思路是将故事存到数据库中（本文用sqlite），然后通过text2sql技术实现自然语言到数据库查询的链接。更具体的流程是：小朋友让讲个故事（或者讲个关于什么什么的故事），大模型识别到是让讲故事，就调用讲故事的工具，这个讲故事的工具是一个基于LLM的text2sql工具，将小朋友的自然语言指令翻译成sql指令进行数据库的查询，然后查询到的故事返回给最初的LLM，实现LLM基于素材库讲故事而不是完全即兴讲故事，当然如果数据库中没有小朋友想听的故事，那么依旧即兴创作。

本文将从数据库基础讲起，逐步实现上述功能。
